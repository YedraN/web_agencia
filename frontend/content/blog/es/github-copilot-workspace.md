GitHub Copilot Workspace es la evolución natural de Copilot. Mientras que Copilot original te ayudaba con fragmentos de código, Workspace entiende issues completos y te guía desde la planificación hasta el pull request.

## ¿Qué es Copilot Workspace?

Copilot Workspace es un entorno de desarrollo asistido por IA que entiende repositorios completos. Puedes asignarle un issue de GitHub y te ayudará a entender el problema, planificar la solución e implementarla.

Funciona analizando la estructura del repositorio, el código existente y el historial de cambios para proponer implementaciones coherentes con la arquitectura del proyecto.

## Flujo de trabajo

**1. Análisis del issue**: Copilot Workspace lee el issue de GitHub, lo desglosa y propone un plan de implementación. Puedes revisar y ajustar el plan antes de proceder.

**2. Generación de código**: Implementa los cambios necesarios en los archivos identificados. Entiende las convenciones del proyecto y genera código consistente.

**3. Revisión**: Muestra los cambios propuestos como diff, permitiéndote revisar cada modificación antes de aceptarla.

**4. Pull request**: Crea un PR con descripción, resumen de cambios y pruebas sugeridas.

## Ventajas sobre Copilot tradicional

Workspace entiende el contexto completo del repositorio, no solo el archivo abierto. Puede trabajar con issues complejos que requieren cambios en múltiples archivos, y entiende la arquitectura general del proyecto.

## Mejores prácticas

Define bien los issues en GitHub. Un issue bien escrito con criterios de aceptación claros produce mejores resultados. Revisa siempre el plan antes de ejecutar. Workspace es una herramienta de productividad, no un reemplazo del desarrollador.

## Disponibilidad

Copilot Workspace está disponible para desarrolladores individuales y equipos, incluido en los planes Copilot Enterprise y como beta gratuita limitada.
